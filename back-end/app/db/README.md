# Signa Database Schema

19 tables on Supabase (PostgreSQL). Run `schema.sql` in the SQL Editor to create/update.

## Per-User Tables

These tables have a `user_id` column and are filtered by the authenticated user.

| Table | Purpose |
|-------|---------|
| **watchlist** | Tickers the user is tracking. Used to trigger Telegram SELL/AVOID alerts and feed the virtual portfolio. Each user has their own watchlist. |
| **portfolio** | User's real holdings — symbol, shares, avg cost, account type (TFSA/RRSP/TAXABLE). For manual portfolio tracking. |
| **positions** | Open/closed trade positions with entry/exit prices, P&L, stop loss, and target. Linked to signals. Used for position monitoring alerts (stop loss hit, target reached, signal weakening). |
| **alerts** | Telegram notifications sent to the user — GEM alerts, watchlist SELL alerts, scan digests, stop loss/target alerts. Tracks delivery status (SENT/FAILED/PENDING). |
| **trade_outcomes** | Recorded trade results for the self-learning loop. Entry/exit prices, P&L, whether signal was correct. Fed to Claude weekly for brain improvement suggestions. |
| **user_settings** | Per-user preferences — theme (midnight/arctic/sunset/ocean/forest/lavender) and language (en/pt). Synced between devices on login. |

## Global Tables

Shared across all users. No user_id filtering.

| Table | Purpose |
|-------|---------|
| **signals** | Scan results — every signal generated by the brain. Contains action (BUY/HOLD/SELL/AVOID), score, target price, stop loss, AI reasoning, technical/fundamental/sentiment data as JSONB blobs. One row per ticker per scan. |
| **scans** | Scan history — type (PRE_MARKET/MORNING/PRE_CLOSE/AFTER_CLOSE), status (RUNNING/COMPLETE/FAILED), progress percentage, phase, tickers scanned, signals found, GEMs found, market regime, duration. Polled by frontend for real-time progress. |
| **tickers** | The 188+ ticker universe — symbol, name, exchange, bucket (SAFE_INCOME/HIGH_RISK), active status. Currently populated from hardcoded universe, could move to DB-driven. |
| **investment_rules** | Brain scoring rules — 38 rules covering solvency, sell triggers, buy conditions, blockers. Each has name, description, formula, thresholds, weights for Safe/Risk buckets, blocker flag. Edited via Brain Editor with 2FA. |
| **signal_knowledge** | Brain knowledge entries — 34 concepts the AI uses during synthesis. Topics like score ranges, GEM conditions, market regime detection, contrarian sentiment. Injected into the Claude prompt as context. |
| **brain_suggestions** | Self-learning output — generated when Claude analyzes trade outcomes. Each suggestion proposes a rule change (adjust threshold, change weight) with reasoning and expected impact. User approves/rejects/applies via Brain Editor. |
| **virtual_trades** | Brain accuracy tracking — automatic paper trades based on scan signals. When a watchlisted ticker gets BUY, the brain "buys" virtually. When it gets SELL/AVOID, it "sells" and calculates P&L. Two sources: "watchlist" (user's picks) and "brain" (auto-picks score 72+). Powers the win rate metric. |
| **ai_usage** | API cost tracking — logs every Claude/Grok/Gemini call with provider, type (synthesis/sentiment), ticker, estimated cost, success/failure. Powers the budget system (daily/monthly limits per provider), the AI Cost stat on the dashboard, and the budget widget. |

## Auth Tables

| Table | Purpose |
|-------|---------|
| **users** | User accounts — username, bcrypt password hash, Telegram chat ID for OTP. |
| **otp_codes** | One-time passwords for login 2FA — code hash, session token, expiry, attempt count. Expires after 2 minutes. |
| **token_blacklist** | Revoked JWT tokens — checked on every authenticated request. Populated on logout. |
| **brain_sessions** | Brain Editor 2FA sessions — OTP hash, brain token JTI, expiry. Separate from login OTP. 15-minute sessions. |
| **audit_logs** | Security event log — login attempts, OTP verifications, brain access, config changes, unauthorized access. Has user_id for tracing but not filtered per-user in normal operation. |

## Relationships

```
users ──┬── otp_codes
        ├── token_blacklist
        ├── brain_sessions
        ├── watchlist
        ├── portfolio
        ├── positions ──── signals
        ├── alerts ─────── signals
        ├── trade_outcomes ── signals
        └── user_settings

signals ──── scans
             tickers

investment_rules  (standalone)
signal_knowledge  (standalone)
brain_suggestions (standalone)
virtual_trades    (standalone)
ai_usage          (standalone)
audit_logs        (standalone)
```

## Key Indexes

- `signals`: by symbol+date, scan_id, score, GEM flag
- `scans`: by status, started_at
- `watchlist`: by user_id, unique(user_id, symbol)
- `positions`: by status (OPEN), symbol
- `virtual_trades`: by symbol, status, source
- `ai_usage`: by provider+date, date
- `audit_logs`: by event_type, created_at
- `brain_suggestions`: by status, analysis_date

## Table Count

19 tables total: 5 auth + 6 per-user + 8 global
